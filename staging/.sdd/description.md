# 機能開発の概要

## 計算問題（クイズ）の回答・判定仕様の変更

### 目的
ユーザーが計算問題を解く際、スクワット動作中（回答選択から立ち上がり完了まで）に回答を変更できるようにし、答え合わせのタイミングを「立ち上がり完了時（回答表示時）」に統一することで、操作性と納得感を向上させる。

### 機能概要
- **回答の選択**:
    - ボタン押下時は「選択状態」とし、即座に正誤判定を行わない。
    - 正解発表（UPフェーズ開始）までは、何度でも選択し直すことができる。
    - 選択中のボタンは視覚的にわかるように強調表示する（例: 枠線、背景色変化など）。
- **答え合わせ（判定）**:
    - フェーズが `UP`（立ち上がり/回答表示）に切り替わったタイミングで判定を行う。
    - 選択されていた回答が正解なら「正解」、不正解なら「不正解」とする。
    - 何も選択されていなかった場合は「不正解」として扱う。
- **UI表示**:
    - 回答ボタンは常時表示（1x4レイアウト）を維持する。
    - 判定時にボタンの色を変化（緑/赤）させ、結果を明示する。
