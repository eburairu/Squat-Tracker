# 機能開発の概要

## お邪魔モード（Disruptive Mode）の仕様変更

### 目的
「お邪魔モード」において、スクワットを行うユーザー（プレイヤー）と計算問題を解くユーザー（妨害者）が異なるという想定に基づき、妨害者が正解することでプレイヤーの進行を阻止（ブロック）できる仕様に変更する。

### 仕様詳細

1.  **モードの定義**
    *   **協力モード (Cooperative)**: 計算問題に正解するとプレイヤーに有利な効果（攻撃ボーナス）が発生する。
    *   **お邪魔モード (Disruptive)**: 計算問題の結果によってプレイヤーの進行（Repカウント）が制御される。

2.  **お邪魔モードの挙動**
    *   **正解時（ブロック成功）**:
        *   妨害者が正解を選んだ場合、プレイヤーへの妨害が成功したとみなす。
        *   **結果**: その回のスクワット（Rep）はカウントされない（無効）。プレイヤーは同じ回をやり直す必要がある。
        *   **フィードバック**: 「ブロック成功！進捗を阻止しました」等のメッセージを表示する。
    *   **不正解時（ブロック失敗）**:
        *   妨害者が不正解を選んだ（または回答しなかった）場合、妨害は失敗したとみなす。
        *   **結果**: その回のスクワット（Rep）は通常通りカウントされ、次の回へ進む。
        *   **フィードバック**: 「ブロック失敗...進捗を許しました」等のメッセージを表示する。

3.  **判定タイミング**
    *   既存の仕様通り、フェーズが `UP`（立ち上がり）に完了した時点で判定を行う。

### 変更点まとめ
| 条件 | 変更前 | 変更後 |
| :--- | :--- | :--- |
| **正解時** | カウントされる（通常進行） | **カウントされない（足止め）** |
| **不正解時** | カウントされない（足止め） | **カウントされる（通常進行）** |
