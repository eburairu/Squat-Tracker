# 実績バッジシステム (Achievement Badges)

## 概要
ユーザーの継続的な利用とモチベーション向上を目的として、特定の条件を満たすと「バッジ」を獲得できるシステムを導入する。
既存の「過去の実績」カードをタブ化し、活動記録とバッジコレクションを切り替えて表示できるようにする。

## 目的
- ユーザーに達成感を提供し、離脱を防ぐ。
- ゲーム要素（RPGモード）との相乗効果を高める。

## 機能要件
- 30種類のバッジ（継続、累計、討伐、特殊条件）。
- バッジの獲得判定はワークアウト完了時に自動で行う。
- 過去のデータ（履歴、討伐数など）に基づいて、初回ロード時または完了時に遡ってバッジを付与する。
- 獲得時にファンファーレ等の演出を行う。
- 「過去の実績」カード内にタブUIを実装し、「記録」と「バッジ」を切り替え可能にする。
- バッジはアイコン（絵文字）と名前、説明文で構成され、未獲得時はグレーアウト表示とする。

## データ構造
- `AchievementSystem` シングルトンで管理。
- `localStorage` キー: `squat-tracker-achievements` (獲得済みバッジIDのリストと日時)。
