# 仕様書: ダイナミック・ボイス・ペルソナ (Voice Persona)

## 1. 目的
ユーザーが音声コーチの「性格（ペルソナ）」を選択できるようにし、トレーニング中のフィードバック（音声とテキスト）をパーソナライズする。これにより、毎日のトレーニングにおけるモチベーション向上と飽きの防止を図る。

## 2. ユースケース
- **ユーザー**は設定画面から好みの音声コーチ（例：標準、熱血、クール、AIなど）を選択できる。
- 選択されたペルソナに応じて、ワークアウト中の「開始」「しゃがむ」「キープ」「立つ」「休憩」などの合図の**発話内容**が変わる。
- 選択されたペルソナに応じて、Web Speech APIの `pitch`（声の高さ）と `rate`（発話速度）が調整され、キャラクターの個性が表現される。

## 3. 機能要件

### 3.1 ペルソナの定義
- ペルソナは外部JSONファイル（`js/data/personas.json`）で定義する。
- 定義するプロパティ:
  - `name`: UIに表示する名前（例: "熱血コーチ"）
  - `pitch`: 声の高さ（基本は `1.0`）
  - `rate`: 声の速度（基本は `1.0`）
  - `lines`: アクションキーに紐づくセリフ群（`start`, `down`, `hold`, `up`, `rest` など）

### 3.2 `VoiceCoach` モジュールの拡張
- `init()` 時点で非同期に `personas.json` をロードする。
- `setPersona(id)` メソッドでアクティブなペルソナを切り替える。
- 従来の `speak(text)` は後方互換性（CommentaryManager等からの呼び出し）のため残し、ペルソナの `pitch` と `rate` だけを適用する。
- 新規に `play(actionKey, defaultText)` メソッドを追加し、ペルソナ辞書に `actionKey` が存在する場合はそのテキストを、存在しない場合は `defaultText` を使用して `speak()` を実行する。

### 3.3 UIとデータ永続化
- 設定モーダル内にペルソナ選択用の `<select>` 要素を追加する。
- ユーザーの選択状態は `localStorage` に保存する。キーは `squat-tracker-voice-persona` を使用する。
- アプリ起動時に保存されたペルソナを復元し、`<select>` の初期値とする。

## 4. エッジケース・失敗時の挙動
- Web Speech API がサポートされていないブラウザでは、音声は再生されない（既存仕様に準ずる）。
- `personas.json` のロードに失敗した場合、デフォルトのペルソナ（またはフォールバック）が適用されること。
- 選択されたペルソナに特定の `actionKey` が定義されていない場合は、引数で渡された `defaultText` を発話すること。

## 5. データ構造 (personas.json 例)
```json
{
  "default": {
    "name": "標準 (Default)",
    "pitch": 1.0,
    "rate": 1.0,
    "lines": {
      "start": "スタートします",
      "down": "しゃがんで",
      "hold": "キープ",
      "up": "立って",
      "rest": "休憩です。深呼吸しましょう"
    }
  },
  "hotblooded": {
    "name": "熱血 (Hot-blooded)",
    "pitch": 0.8,
    "rate": 1.2,
    "lines": {
      "start": "気合入れていけ！スタートだ！",
      "down": "もっと深く！",
      "hold": "そこで耐えろ！",
      "up": "よし、立て！",
      "rest": "休んでる暇はないぞ！息を整えろ！"
    }
  }
}
```
