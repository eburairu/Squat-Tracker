# Spec-018: バディ育成・進化システム (Buddy Evolution)

## 1. 概要
既存の「バディシステム」を拡張し、経験値によるレベルアップと、特定条件での「進化」機能を実装する。
ユーザーのトレーニング量（スクワット回数）をバディの成長に還元することで、継続的なモチベーションと愛着を醸成する。

## 2. 目的
- **「育てる」楽しさの提供**: 単なる数値強化だけでなく、見た目や名前の変化（進化）により、視覚的な報酬を与える。
- **継続率の向上**: 「あと少しでレベルアップ/進化する」という動機付けにより、日々のトレーニングを促進する。

## 3. 機能要件

### 3.1 経験値 (EXP) 獲得
- **獲得条件**: ワークアウト完了時 (`finishWorkout`)。
- **計算式**: `獲得EXP = 合計スクワット回数 (Total Reps)`
  - 例: 3セット×10回 = 30 EXP
- **対象**: 現在「装備中」のバディのみ獲得。

### 3.2 レベルアップ (Level Up)
- **必要EXP**: `Next Level * 50` (累積ではなく、レベルごとの必要量)
  - Lv.1 -> Lv.2: 100 EXP
  - Lv.2 -> Lv.3: 150 EXP
- **効果**:
  - 攻撃力ボーナスが増加。
  - 計算式: `Base Damage + (Level * 2)`
- **演出**:
  - レベルアップ時にトースト通知 (`showToast`) を表示。
  - 「🎉 バディがレベルアップ！ Lv.1 -> Lv.2」

### 3.3 進化 (Evolution)
- **条件**: 特定のレベルに到達し、進化データが存在する場合。
- **データソース**: `js/data/buddy-evolution.json`
- **効果**:
  - 名前 (`name`) とアイコン (`emoji`) が変化。
  - 基礎ダメージ (`baseDamage`) が上昇。
  - レベルは `1` にリセットされる（強くてニューゲーム形式）。
- **演出**:
  - 進化時に特別なトーストまたはモーダル表示。
  - 「✨ スライムは キングスライム に進化した！」

### 3.4 UI表示拡張
- **装備モーダル (`#equipment-modal`)**:
  - バディリストに `Lv.{level}` を表示。
  - 経験値バー（プログレスバー）を追加し、次のレベルまでの進捗を表示。
- **リザルト画面**:
  - バディの獲得経験値を表示（トースト等で簡易的に）。

## 4. データ構造変更

### 4.1 Buddy Object (拡張)
```json
{
  "id": "slime",
  "name": "スライム",
  "emoji": "💧",
  "level": 1,
  "exp": 0,
  "baseDamage": 3,      // 追加: 進化で変動するため
  "acquiredAt": 1715400000000
}
```

### 4.2 Evolution Data (`js/data/buddy-evolution.json`)
```json
{
  "スライム": {
    "targetLevel": 5,
    "nextId": "king_slime",
    "nextName": "キングスライム",
    "nextEmoji": "👑",
    "baseDamageBonus": 5
  }
}
```

## 5. 設計詳細

### 5.1 BuddyManager
- `addExp(amount)`: 装備中のバディにEXPを加算し、レベルアップ・進化判定を行う。
- `checkLevelUp(buddy)`: EXPが閾値を超えたか判定。
- `evolve(buddy, evolutionData)`: バディを進化形態に更新する。
- `getRequiredExp(level)`: 必要EXPを計算。

### 5.2 互換性対応
- 既存のセーブデータ（`baseDamage` プロパティがない）を読み込んだ際、デフォルト値（`3`）で補完するマイグレーションロジックを入れる。

## 6. テスト計画
- **Unit Test (Playwright)**:
  - `BuddyManager` を直接操作し、EXP加算、レベルアップ、進化が正しく行われるか検証。
- **E2E Test**:
  - ワークアウト完了後、バディのEXPが増えていることを確認。
  - 進化条件を満たした際、UI上の名前とEmojiが変わることを確認。
