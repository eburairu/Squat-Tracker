# 仕様書: フォーム・アナライザー（テンポ・リズム評価システム）

## 目的
ユーザーが設定したスクワットのテンポ（しゃがむ時間、キープ時間、立つ時間）にどれだけ正確に追従できたかを計測し、セッション終了後に評価（スコアとランク）としてフィードバックする。これにより、ただ回数をこなすだけでなく「正しいフォーム（テンポ）で運動できているか」という運動の質の向上をサポートする。

## 機能要件
1. **フェーズ時間の計測**: `DOWN`, `HOLD`, `UP` 各フェーズの実際の経過時間を計測する。
2. **精度計算**: 設定された理想の各フェーズ時間と実際の経過時間との差異から「テンポ精度（%）」を算出する。
3. **ランク評価**: 算出された総合精度（スコア）に基づき、ランク（S, A, B, C）を決定する。
4. **結果の表示**: セッション終了時（`finishWorkout`）に、算出されたスコアとランクをトースト通知、またはリザルト画面に表示する。
5. **ヒストリーへの保存**: （可能であれば）`historyEntries` の各エントリにスコア情報を付与し、後から振り返れるようにする。（今回はセッションごとの評価表示をMVPとする）

## 非機能要件
- **軽量な処理**: 計測ロジックは既存のタイマー処理を妨げず、パフォーマンスに影響を与えないこと。
- **拡張性**: 将来的に各フェーズ（DOWN/HOLD/UP）ごとの詳細なフィードバックを行えるようなデータ構造にすること。

## 想定ユースケース
1. ユーザーが「ダウン3秒・キープ2秒・アップ1秒」でセッションを開始する。
2. ユーザーはガイドに合わせてスクワットを行うが、少し早めに立ち上がってしまう。
3. セッション終了後、「テンポ精度: 85% (Rank A)」という評価が表示され、次回はもっとキープ時間を意識しようと気づく。

## 入出力仕様
- **入力**:
  - `targetDurations`: `{ down: number, hold: number, up: number }` (各フェーズの設定時間 ms)
  - `actualDurations`: 1レップごとの実測時間データの配列 `[{ down: number, hold: number, up: number }, ...]`
- **出力**:
  - `score`: `number` (0〜100 のパーセンテージ)
  - `rank`: `string` ('S', 'A', 'B', 'C')
  - `feedbackMessage`: `string` (結果に応じた短いコメント)

## エッジケース
- 途中で一時停止（Pause）した場合: 一時停止していた時間は実測時間から除外するか、一時停止があったレップの計測をスキップまたは緩和する。（MVPでは、極端に長い時間はペナルティとして扱うか、許容誤差の範囲で処理する）
- センサーモード使用時: センサーによる入力でフェーズが切り替わるため、実測時間と設定時間のブレが生じやすい。評価基準をセンサーモードと通常モードで分けるか検討（MVPでは同一基準で試験導入）。

## 失敗時の挙動
- 何らかの理由で計測データが空の場合（例：エラーで途切れた）、評価計算をスキップし、スコア・ランクの表示を行わない。アプリ本体の進行（セーブ処理や報酬付与）には影響を与えないこと。
