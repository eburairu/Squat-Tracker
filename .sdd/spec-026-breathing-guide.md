# 仕様書: 呼吸ガイド・システム (Breathing Guide System)

## 1. 目的 (Purpose)
ユーザーがスクワットの動作に合わせて「正しい呼吸」を行えるよう、視覚的なフィードバックを提供する。
正しい呼吸（しゃがむ時に吸い、立つ時に吐く）は、血圧の急上昇を防ぎ、体幹を安定させ、トレーニングの安全性と効果を高める。
現在のアプリは「動き（Down/Hold/Up）」のガイドしか提供していないため、これに「呼吸（Inhale/Hold/Exhale）」の概念を追加し、ヘルスケア志向の機能として価値を提供する。

## 2. 機能要件 (Functional Requirements)
- アプリのタイマーフェーズ（DOWN, HOLD, UP）に同期して、呼吸の指示を視覚的に表示する。
  - **DOWN (しゃがむ)**: 「吸う (Inhale)」
  - **HOLD (キープ)**: 「止める (Hold)」
  - **UP (立つ)**: 「吐く (Exhale)」
- 呼吸ガイドの表示ON/OFFを切り替える設定トグルを提供する。
- 呼吸ガイドの有効/無効状態を `localStorage` に保存・復元する。
- 呼吸を直感的に表現するアニメーション（例：肺が膨らむ/縮むようなエフェクト、円が拡大/縮小する等）を実装する。

## 3. 非機能要件 (Non-Functional Requirements)
- UIは既存のトレーニング画面（円形のプログレスバーやタイマー表示）に違和感なく溶け込むデザインとする。
- パフォーマンスへの影響を最小限にする（CSSアニメーションを活用する）。
- 直感的にわかるアイコンやテキストを使用する。
- デフォルトはOFFとし、ユーザーが任意で有効化できるようにする。

## 4. 想定ユースケース (Use Cases)
- **ケース1**: 初心者がスクワットを始める際、呼吸のタイミングがわからないため、「呼吸ガイド」をONにしてトレーニングを行う。
- **ケース2**: 安全性を重視したいユーザーが、正しいフォームと呼吸を意識するためにガイドを利用する。
- **ケース3**: 上級者が不要と感じた場合はOFFにして、通常のカウントのみでトレーニングを行う。

## 5. 入出力仕様 (I/O Specifications)
- **入力**:
  - 設定パネルの「呼吸ガイド」トグルの切り替え操作。
  - アプリケーション本体 (`app.js`) からのフェーズ変更イベント (`Phase.DOWN`, `Phase.HOLD`, `Phase.UP`)。
- **出力**:
  - タイマー画面内への呼吸指示テキスト（例：「大きく吸う」）の表示。
  - 呼吸に合わせた視覚的なエフェクト（CSSクラスの切り替え）。
- **データ保存**:
  - `localStorage` キー: `STORAGE_KEYS.BREATHING_GUIDE` (Boolean)

## 6. エッジケース (Edge Cases)
- 休憩フェーズ (`Phase.REST`) や待機中 (`Phase.IDLE`) では、呼吸ガイドの特別な指示は出さず、デフォルトの「自然な呼吸」などのニュートラルな状態にするか非表示にする。
- センサーモードや音声コマンド操作中でも、呼吸ガイドは正常に画面に表示されること。

## 7. 失敗時の挙動 (Error Handling)
- `localStorage` からの読み込みに失敗した場合は、デフォルト設定（OFF）として扱う。
- フェーズの変更イベントが正しく受け取れなかった場合でも、アプリ自体の動作（カウント等）は停止せず、呼吸ガイドのみが更新されない状態に留める。
