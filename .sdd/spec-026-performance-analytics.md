# Specification: パフォーマンス・アナリティクス（自動評価＆フィードバック）

## 1. 目的
単なる「回数消化」ではなく、「質の高いトレーニング」を促すこと。
セッション終了時にフォーム（テンポの正確性）や達成度を自動評価し、ランク付けと改善アドバイスを提示することで、ユーザーが次回のトレーニングで意識すべきポイントを明確にする。

## 2. 機能要件
- **スコア計算**:
  - 各レップの各フェーズ（Down, Hold, Up）の所要時間を、設定された目標時間（テンポ設定）と比較し、正確性を算出する。
  - セッション全体の「テンポ正確率（％）」を算出する。
  - 設定された目標回数・セット数に対する実際の達成率を加味し、総合スコア（0〜100点）を算出する。
- **ランク判定**:
  - Sランク: 95〜100点
  - Aランク: 80〜94点
  - Bランク: 60〜79点
  - Cランク: 60点未満
- **フィードバック生成**:
  - 最も誤差が大きかったフェーズ（例: しゃがむのが早すぎた、キープが短すぎた等）を特定し、日本語で具体的なアドバイス文を生成する。
- **UI表示**:
  - セッション完了時に表示されるモーダル（リザルト画面）に、ランク（S/A/B/C）、総合スコア、フィードバックコメントを表示する領域を追加する。

## 3. 非機能要件
- 既存のワークアウトフロー（タイマー、セッション完了処理）を阻害しないこと。
- スコア計算処理は軽量であること。
- テスト可能であること。

## 4. 想定ユースケース
1. ユーザーが「スクワット 10回 × 3セット」を開始する。
2. ユーザーはリズムガイドに合わせてスクワットを行うが、少し早まりがちになる。
3. セッション終了。
4. リザルト画面が表示され、「ランクB: 75点。全体的にペースが早まっています。『しゃがむ』フェーズをもっとゆっくり行いましょう。」とフィードバックされる。
5. ユーザーは次回のトレーニングでペースを意識するようになる。

## 5. 入出力仕様
- **入力**:
  - セッション情報（目標レップ、目標セット、各フェーズの目標秒数）
  - 実際の経過データ（FormAnalyzer または WorkoutTimer から収集した各レップの各フェーズの実測秒数）
- **出力**:
  - 総合スコア (number: 0-100)
  - ランク (string: 'S', 'A', 'B', 'C')
  - フィードバックメッセージ (string)

## 6. エッジケース
- 目標秒数が設定されていない、または0秒のフェーズがある場合（エラーにならず、評価対象から除外する）。
- 途中でリタイアした場合（達成率が下がるため、それに応じたスコア・ランクとなる）。
- Active Recovery モードなど、特殊なモードでのプレイ時（必要であれば評価をスキップする）。

## 7. 失敗時の挙動
- データ収集に失敗した場合、またはデータが不十分な場合（1レップも完了していない等）、評価はスキップされるか「評価不能」として表示しない。
