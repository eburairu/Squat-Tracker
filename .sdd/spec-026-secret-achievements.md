# シークレット実績機能 (Secret Achievements)

## 1. 目的

現在の実績（バッジ）システムでは、未獲得の実績もその達成条件や名前が最初から全て表示されている。
ユーザーの好奇心を刺激し、「何か隠された条件があるのでは？」と様々な機能を試行錯誤させるため、特定の条件を満たすまで内容が伏せられる「シークレット実績」を導入する。
これにより、ユーザーに驚きと探索の楽しみを提供し、モチベーションの向上とアプリの継続利用を促す。

## 2. 機能要件

- **シークレットフラグの追加**:
  - 実績データ（`js/data/achievements.json`）の各項目に対して、オプショナルな `secret` プロパティ（boolean型）を設定できるようにする。
- **未獲得時のマスキング表示**:
  - `secret` が `true` かつ未獲得の実績の場合、実績一覧のグリッド表示において以下のように情報を隠す。
    - アイコン（emoji）: 「❓」
    - 実績名（name）: 「???」
  - 実績をクリックした際に表示されるアラート（詳細情報）においても、以下のように隠す。
    - アイコン: 「❓」
    - 実績名: 「???」
    - 説明文（description）: 「条件は秘密です」
- **獲得時の通常表示**:
  - 対象の実績を獲得（アンロック）した後は、通常の実績と同様に本来のアイコン、名前、説明文が表示される。

## 3. 非機能要件

- 既存の実績システム（シークレットではない実績）の表示や判定ロジックに影響を与えないこと。
- データ構造の変更は最小限に留め、既存の `achievements.json` との互換性を保つこと。

## 4. 想定ユースケース

1. ユーザーが実績タブを開く。
2. グリッドの中にアイコンが「❓」で名前が「???」の実績を見つける。
3. クリックしても「条件は秘密です」と表示され、どうすれば獲得できるかわからない。
4. ユーザーがアプリの様々な機能（深夜のスクワット、特定の回数など）を試す。
5. 偶然または意図的に条件を満たし、「実績解除！」の通知とともに本来の名前とアイコンが表示される。
6. 再度実績タブを開くと、該当の実績がアンロックされ、詳細が確認できるようになる。

## 5. 入出力仕様

- **入力**:
  - `js/data/achievements.json` 内の各実績オブジェクトにおける `"secret": true`。
- **出力**:
  - UIへの描画時（`js/modules/achievement-system.js` の `render` メソッド）における、マスキングされたテキスト・アイコンの出力。

## 6. エッジケースと失敗時の挙動

- `secret` プロパティが存在しない既存の実績については、デフォルトで `secret: false` 扱いとし、通常通り表示されること。
- 実績獲得通知（`showNotification`）については、獲得した瞬間であるため、本来のアイコンと名前が表示されること（通知で「???」とならないようにする）。これは既存ロジックで条件達成時に `showNotification(badge)` が呼ばれるため、`badge` オブジェクトの生データを使えば問題ない。
