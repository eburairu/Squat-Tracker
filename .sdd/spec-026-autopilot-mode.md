# オートパイロット・モード（おまかせスタート）

## 1. 目的

ユーザーがアプリを開いてからトレーニングを開始するまでの「設定や状況判断による認知負荷（摩擦）」を最小化する。
本日のミッション状況、運勢、装備状況をシステムが自動で最適化し、1タップで最適なワークアウトを開始できる「ワンタップ・オートパイロット」機能を提供する。

## 2. 機能要件

### 2.1 自動最適化フロー (`AutopilotManager`)

`startOptimalSession()` 実行時に以下の処理を自動で連鎖的に行う。

1. **運試し（Fortune）**
   - 本日の運勢を引いていない場合は、`FortuneManager.draw()` を実行してボーナスを確定させる。
2. **最強装備の適用（Loadout）**
   - `LoadoutManager.optimizeLoadout()` を実行し、現在所持しているクラス・武器・称号から最も攻撃力の高くなる組み合わせを自動で装備する。
3. **最適なメニューの算出と設定（Smart Mission Routine）**
   - `SmartMissionRoutine.calculateOptimalRoutine()` を使用して、未達のデイリーミッションおよびビンゴを達成するために必要な「セット数」と「回数」を算出する。
   - 算出された値で画面上の `<input id="set-count">` および `<input id="rep-count">` の値を更新し、変更イベント（`input`, `change`）を発火させてシステムの内部状態に同期させる。
   - すべて達成済みの場合は現在の設定を維持する。
4. **ワークアウトの自動開始**
   - 上記の設定完了後、少しの遅延（約500ms、視覚的フィードバックとトースト表示をユーザーが認識する時間）を挟んだのち、既存の `startWorkout()` コールバックを実行してカウントダウンフェーズに移行する。

### 2.2 UI追加

- メインのスタートボタン（`#start-button`）に隣接する形で、「✨ おまかせ」ボタン（`#autopilot-start-button`）を配置する。
- ボタン押下時にトースト通知などで「自動最適化完了」のフィードバックを与える（一部は既存モジュールのトーストを活用）。

## 3. 非機能要件

- 既存の `FortuneManager`, `LoadoutManager`, `SmartMissionRoutine` モジュールに手を加えず、それらを統括するファサード（Facade）パターンとして `AutopilotManager` を実装する。
- 処理に失敗した場合（各モジュールの予期せぬエラーなど）でも、進行を妨げずに通常のワークアウト開始処理（フォールバック）が行われること。

## 4. 想定ユースケース

- **シナリオA（忙しい朝）**
  1. ユーザーがアプリを起動する。
  2. 「✨ おまかせ」ボタンをタップする。
  3. 自動的に「大吉」が引かれ、最強装備がセットされ、今日のミッション達成に必要な「3セット×12回」がセットされて即座にトレーニングが始まる。

## 5. 入出力仕様

### `AutopilotManager.startOptimalSession(startWorkoutCallback)`

- **入力**:
  - `startWorkoutCallback`: {Function} 全ての準備が完了した後に呼ばれる、ワークアウト開始関数（`js/app.js` の `startWorkout`）。
- **出力**:
  - なし（非同期でコールバックを実行）

## 6. エッジケースとエラーハンドリング

- `SmartMissionRoutine.calculateOptimalRoutine()` が `null` を返した場合（ミッション全達成済み）：
  - セット数・回数の上書きは行わず、ユーザーの現在の設定値（またはデフォルト値）のままワークアウトを開始する。
- 各モジュールが未初期化、またはエラーをスローした場合：
  - `try-catch` で捕捉し、最低限のエラーログを出力した上で `startWorkoutCallback()` を実行し、ユーザー体験を損なわないようにする。
