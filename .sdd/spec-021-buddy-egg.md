# バディ・エッグ (Buddy Egg) 機能仕様書

## 1. 概要
ユーザーが日々のスクワット回数を蓄積することで「タマゴ（Egg）」を孵化させ、新しいバディ（コンパニオン）や報酬を獲得できる機能。
歩数計アプリの「歩いてタマゴを孵す」体験をスクワットに置き換え、毎日のトレーニングに視覚的な目標と「あと少しで孵る」という強力な動機付けを提供する。

## 2. 目的
- **継続への動機付け**: 「あと〇回で孵化する」という明確なゴールを提示し、毎日のトレーニング回数を増やす、あるいはサボりを防ぐ。
- **コレクション要素の強化**: ランダムな報酬（バディ）を得るワクワク感を提供し、図鑑埋めや育成のモチベーションを高める。
- **視覚的なフィードバック**: 成長過程（ヒビが入る、光るなど）を可視化し、達成感を高める。

## 3. 機能要件

### 3.1 タマゴの取得と状態管理 (`EggManager`)
- ユーザーは常に1つの「アクティブなタマゴ」を持つ（初期リリースでは1スロットのみ）。
- タマゴの状態は `localStorage` の `squat-tracker-egg` キーで管理する。
- 状態データ構造:
  - `id`: タマゴの一意識別子
  - `type`: タマゴの種類（`basic`, `rare`, `epic` など。今回は `basic` のみ）
  - `requiredReps`: 孵化に必要な総スクワット回数（例: 100回、500回など）
  - `currentReps`: 現在蓄積されているスクワット回数
  - `status`: 状態（`incubating`: 孵化待ち, `ready`: 孵化可能, `hatched`: 孵化済み）

### 3.2 進捗の蓄積 (`addProgress`)
- 毎回のスクワット完了時（`finishWorkout` 時）に、実行した回数（`totalReps`）をタマゴの `currentReps` に加算する。
- `currentReps` が `requiredReps` に達した場合、状態を `ready`（孵化可能）に変更する。

### 3.3 孵化ロジック (`hatchEgg`)
- 状態が `ready` のタマゴをクリックする、または自動的にポップアップして孵化イベントを発生させる。
- 孵化時に、ランダムな報酬（今回は既存のバディからランダムに1体、または `BuddyManager.addExp` 用の経験値アイテムなど）を付与する。
  - 初期リリースでは、既存のバディリスト（`js/data/buddies.json` があればそれ、なければランダムな名前とアイコンを生成）から1体を獲得するか、既存の `BuddyManager` に連携する。
- 孵化が完了すると、次の新しいタマゴが自動的にセットされる（ループ構造）。

### 3.4 UI/UX
- **メイン画面**: ヘッダー付近、またはバディ/ステータス表示エリアに「タマゴアイコン」と「進捗バー（`currentReps` / `requiredReps`）」を配置する。
- **視覚的変化**: 進捗率に応じてタマゴのアイコンが変化する（0%: 無傷 → 50%: ヒビ → 90%: 大きなヒビ/震え → 100%: 孵化可能エフェクト）。
- **孵化モーダル**: タマゴが孵化する際、専用のモーダルウィンドウ (`#egg-hatch-modal`) を表示し、アニメーション演出（タマゴが割れて中からバディが飛び出す）と共に報酬を提示する。

## 4. 非機能要件
- **永続化**: ブラウザの `localStorage` に確実にデータを保存し、リロードしても進捗が失われないこと。
- **パフォーマンス**: アニメーションや進捗更新が軽量であること。

## 5. 想定ユースケース
1. ユーザーがアプリを開く。「タマゴ孵化まであと 30回」と表示されている。
2. ユーザーが 3セット×10回（計30回）のスクワットを実行する。
3. 完了画面後、タマゴが震え出し、「孵化可能！」の状態になる。
4. ユーザーがタマゴをタップすると、ファンファーレと共に新しいバディ「スライム」が誕生する。
5. 次のタマゴ（目標: 50回）が自動的にセットされる。

## 6. 入出力仕様 (EggManager)

- **入力**:
  - `init()`: アプリ起動時の初期化。DOMのバインド。
  - `addProgress(reps: number)`: 実行回数の追加。
  - `hatch()`: 孵化の実行。
- **出力**:
  - `getEggStatus()`: `{ currentReps, requiredReps, status, type }` を返す。
  - UIの動的更新（プログレスバー、タマゴの画像/絵文字、モーダル表示）。

## 7. エッジケースと失敗時の挙動
- **データ破損**: `localStorage` のデータが不正な場合、エラーをコンソールに出力し、新しい初期タマゴを再生成する。
- **超過分の繰り越し**: 必要な回数を大きく上回ってスクワットをした場合（例: あと10回で孵化するのに50回やった）、残りの40回は「次のタマゴ」に繰り越されること（ユーザーの努力を無駄にしない）。

## 8. テスト方針
- 単体/モジュールテスト（Playwrightを使用）:
  - 初期状態のタマゴが正しくセットされるか。
  - `addProgress` によって進捗が正しく更新されるか。
  - 進捗が100%に達したとき、状態が `ready` になるか。
  - 孵化モーダルが表示され、報酬が取得できるか。
  - 超過した回数が次のタマゴに正しく繰り越されるか。
