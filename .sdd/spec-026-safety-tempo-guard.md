# Spec 026: セーフティ・テンポ・ガード (Safety Tempo Guard)

## 1. 目的
スクワットの動作スピード（テンポ）が速すぎる設定になった場合、ユーザーに怪我のリスクやフォーム崩れの危険性を警告し、安全で効果的なテンポを提案することで、トレーニングの質と安全性を向上させる。

## 2. 機能要件
- **テンポのリアルタイム監視**: ユーザーが設定する「しゃがむ時間 (Down)」「キープ時間 (Hold)」「立つ時間 (Up)」の入力値を監視する。
- **警告表示**: 以下の条件に合致した場合、ワークアウト設定エリアに警告メッセージを表示する。
  - **条件A (Fast Down)**: 「しゃがむ時間」が1秒に設定されている場合。
    - メッセージ例: 「しゃがむスピードが速すぎます！膝への負担を減らすため、2秒以上かけてゆっくりしゃがむことを推奨します。」
  - **条件B (Fast Tempo)**: 動作の合計時間 (Down + Hold + Up) が3秒以下の場合。
    - メッセージ例: 「動作テンポが速すぎます。フォーム崩れや怪我を防ぐため、1回あたり4秒以上かける設定を推奨します。」
  - 条件Aと条件Bの両方に合致する場合は、両方の警告、または優先度の高い警告（条件B）を表示する。
- **安全テンポの提案と適用機能**: 警告表示と同時に、「安全なテンポ(2-1-1)を適用する」ボタンを提示し、ユーザーが1タップで推奨設定（しゃがむ2秒、キープ1秒、立つ1秒）に変更できるようにする。

## 3. 非機能要件
- **即時フィードバック**: 入力値が変更された瞬間に警告の表示/非表示が切り替わること。
- **UIの邪魔をしない**: 警告メッセージは目立つ（例: オレンジや赤のボーダー、アイコン）が、既存のレイアウトを大きく崩さないコンパクトなデザインであること。
- **アクセシビリティ**: 警告が表示された際、スクリーンリーダーなどで読み上げられるよう `aria-live` 属性を適切に設定する。

## 4. 想定ユースケース
1. ユーザーが早くトレーニングを終わらせようと、各フェーズの時間を1秒（1-1-1）に設定する。
2. アプリが即座に「動作テンポが速すぎます...」という警告を表示する。
3. ユーザーが警告に気付き、「安全なテンポを適用する」ボタンをタップする。
4. 設定値が自動的に「しゃがむ2秒、キープ1秒、立つ1秒」に修正され、警告が消える。
5. ユーザーは安全なペースでトレーニングを開始できる。

## 5. 入出力仕様
- **入力**:
  - `downDurationInput` の `value`
  - `holdDurationInput` の `value`
  - `upDurationInput` の `value`
- **出力**:
  - 警告UIのDOM要素の表示/非表示の切り替え
  - 警告メッセージのテキスト更新
  - 推奨設定適用ボタンの表示と、クリック時の各Input要素への値の反映（および関連イベントの発火）

## 6. エッジケース
- 入力欄が空、または不正な値（NaN）になった場合、警告ロジックはエラーを起こさず、評価をスキップするか安全側に倒す（警告非表示）。
- プリセットやスマートプランナー機能によって値が変更された場合も、同様に監視ロジックが走り、不適切な値なら警告を表示すること。

## 7. 失敗時の挙動
- もし警告UIのDOM取得に失敗した場合、機能はサイレントに無効化され、アプリ全体の動作（スクワットの開始など）を妨げないこと。
