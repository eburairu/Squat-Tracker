# 仕様書: ワンタップ・デイリー・ディレクター (One-Tap Daily Director)

## 1. 目的
アプリ起動時にユーザーが「今日は何回やればいいか」「どのミッションが残っているか」「ボスを倒すにはどれくらいのダメージが必要か」を考える認知負荷をゼロにする。ワンタップで最適な「メニュー設定」「最強装備の適用」「トレーニング開始」を全自動で行い、ユーザーの操作ミス（装備忘れ、ミッション漏れ）を防ぎつつ、即座に没入できる体験を提供する。

## 2. 機能要件
1. **コンテキスト統合解析ロジック**
   - 以下のデータを収集・解析する。
     - デイリーミッションおよびビンゴの未達成要件（必要回数・セット数）。
     - ボスバトルの状態（ボスの残りHPと現在クラスでの推定必要回数）。
     - 過去のトレーニング履歴（ベースとなる体力水準）。
2. **最適メニューの算出と提示**
   - 解析結果から「本日の最適メニュー（セット数×回数）」を自動計算する。
   - ヒーローエリア（またはそれに準ずる目立つ位置）に「AIディレクター推奨メニュー」として表示する。
   - 推奨理由（例：「ミッション達成とボス撃破に最適化されています」）を併記し、納得感を持たせる。
3. **ワンタップ実行機能**
   - ユーザーが「推奨メニューで開始」ボタンを押下した際、以下の処理を順次実行する。
     1. メニュー（セット数・回数）のフォームへの反映。
     2. `LoadoutManager.optimizeLoadout()` の呼び出しによる最強装備の自動適用。
     3. ワークアウトの開始（自動的に開始プロセスへ移行）。

## 3. 非機能要件
- **パフォーマンス**: 解析処理は同期/非同期問わず高速に行い、初期ロード時間を体感的に遅延させないこと。
- **UI/UX**: 既存のテーマやデザインシステム（カードUI等）に馴染むデザインであること。
- **モジュール性**: 既存の `SmartMissionRoutine` や `SmartPlanner` のロジックと競合せず、可能ならロジックを再利用・統合すること。

## 4. 想定ユースケース
- **仕事終わりの疲れた夜**: ユーザーはアプリを開き、細かい設定をする気力がない。「今日の推奨：3セット×12回（ミッションとボス撃破に最適）」というパネルを見て、何も考えずに「開始」を押し、そのままトレーニングを終えて報酬を全て獲得する。

## 5. 入出力仕様
- **入力**:
  - `DailyMissionSystem.state`
  - `BingoManager.state`
  - `BossBattle.state`
  - `InventoryManager` (所持武器データ)
  - `history` (ローカルストレージの過去履歴)
- **出力**:
  - `sets` (推奨セット数: 1〜10)
  - `reps` (推奨回数: 5〜50)
  - `message` (推奨理由のテキスト)
  - `action`: ボタン押下イベント時の状態更新とアプリ開始トリガー

## 6. エッジケース
- **ミッション全達成済 ＆ ボス撃破直後**: 過去の履歴に基づいた「現状維持」メニューを提案する。
- **ボスHPが膨大（後半の敵）で必要回数が非現実的**: 1日の上限（例: 最大5セット、1セット最大30回など）を設け、過度な負荷を避ける。
- **初回起動時（履歴なし）**: デフォルトの軽量メニュー（3セット×10回）を提案する。

## 7. 失敗時の挙動
- データの読み込みに失敗、または依存モジュールが初期化されていない場合は、ディレクターパネル自体を非表示（または「データ収集中」表示）とし、通常の手動設定を利用させる。エラーでアプリ全体がクラッシュすることは絶対に避ける。
