# 相棒の卵 (Buddy Egg) システム 仕様書

## 1. 目的
毎日のスクワット回数を蓄積し、一定数に達すると新しいBuddyを獲得できる「相棒の卵」機能を追加する。
これにより、日々のトレーニング回数が「卵の孵化」という明確な目標に結びつき、ユーザーのモチベーション向上と継続を促進する。

## 2. ユーザー体験 (UX)
1. **卵の確認**: メイン画面のステータスパネル内に「バディの卵」の進捗が表示される。
2. **回数の加算**: ワークアウト（スクワット）を完了すると、実施した回数が卵のプログレスバーに加算される。
3. **孵化**: 累計回数が規定値（デフォルト100回）に達すると、卵が孵化するアニメーションとともに通知が表示され、ランダムなモンスターが新しいBuddyとして仲間になる。
4. **次の卵**: 孵化後、超過した回数は次の卵の進捗に引き継がれ、再び新しい卵の育成が始まる。

## 3. 機能要件

### 3.1 データ構造
- `localStorage` キー: `squat-tracker-egg` (`STORAGE_KEYS.EGG` として定義)
- データ形式:
```json
{
  "currentReps": 0,
  "requiredReps": 100,
  "hatchedCount": 0
}
```

### 3.2 コア機能 (EggManager)
- **初期化 (`init`)**: ストレージからデータを読み込み、UIを初期化する。
- **回数加算 (`addReps`)**:
  - 完了したスクワット回数を受け取り、`currentReps` に加算する。
  - `currentReps >= requiredReps` になった場合、孵化処理 (`hatch`) を呼び出す。
  - 超過分を考慮し、複数回連続で孵化する可能性もサポートする（`while` ループなどで処理）。
- **孵化処理 (`hatch`)**:
  - `MONSTERS` リストからランダムにモンスターを選択する。
  - `BuddyManager._forceAdd(monsterIndex)` を用いてBuddyを追加する。
  - 孵化アニメーション（UI上のクラス付与等）とトースト通知を表示する。
  - `hatchedCount` をインクリメントする。

### 3.3 UI要件
- **メイン画面ステータス**:
  - `<div class="stat-card" id="egg-tracker">` として、ステータスグリッドの末尾に配置。
  - 卵の絵文字（🥚）、プログレスバー、テキスト（現在の回数 / 必要な回数）を表示する。
  - 孵化時にはCSSアニメーション(`.hatching`)を再生する。

## 4. 非機能要件
- **永続化**: ブラウザを閉じてもデータが保持されること。
- **堅牢性**: データが破損・未定義であっても、デフォルト値（0 / 100）を使用して安全にフォールバックすること。
- **独立性**: `EggManager` は回数の管理に特化し、Buddyの追加ロジックは既存の `BuddyManager` に依存する。

## 5. エッジケース
- **大量の回数追加**: 一度のワークアウトで必要回数を大きく超える回数（例: 250回）が追加された場合、複数回（この例では2回）孵化処理が行われ、残りの50回が次の卵の進捗に反映されること。

## 6. テスト方針 (Playwright)
- **初期状態**: 初期読み込み時に卵のUI要素が正しく表示されること。
- **プログレスバーの更新**: `EggManager.addReps(10)` を呼び出すと、進捗テキストが `10 / 100` になり、プログレスバーの幅が 10% になること。
- **孵化の検証**: `EggManager.addReps(100)` を呼び出した際、UIにアニメーションクラスが付与され、`BuddyManager` の所持Buddy数が1つ増加していること。
