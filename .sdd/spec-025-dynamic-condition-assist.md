# Specification: Dynamic Condition Assist (ダイナミック・コンディション・アシスト)

## 1. 目的

ユーザーの日々の体調や疲労度をトレーニング中の行動から推測し、過負荷によるトレーニングの挫折や怪我を防ぐ。具体的には、第1セット目の完了後（またはトレーニングの中盤）に、ペースの遅れやクイズの不正解率から「疲労」を検知し、残りのセット数や回数を減らす（負荷を軽くする）提案を行う。これにより、ユーザーに「無理しなくていい」という安心感を与え、継続率（ストリーク）の維持をサポートする。

## 2. 機能要件

- **疲労度検知ロジック**: 第1セット完了時のデータを収集・分析する。
  - **評価指標**:
    1. **一時停止の有無と長さ**: セット中に一時停止（Pause）を行ったか、または一定時間以上ポーズしていたか。
    2. **クイズの正答率**: 出題されたクイズの正答率が著しく低いか（例: 50%未満）。
    3. **ペースの遅れ（ゴーストとの比較）**: 過去の自分（または標準ペース）と比較して、進行が遅いか。
- **アシスト提案のトリガー**:
  - 検知ロジックにより「疲労」スコアが閾値を超えた場合、第1セット完了直後の「休憩フェーズ（Phase.REST）」開始時に、ダイアログ（モーダル）を表示して提案を行う。
  - すでに総セット数が1（または残りセット数が0）の場合は発火しない。
  - 1回のトレーニングセッションにつき1回のみ発火する。
- **提案内容**:
  - 「今日は少しお疲れのようですね。無理せず、残りのセット数を減らしますか？」
  - 選択肢1: 「はい（セット数を1減らす、または回数を減らす）」
  - 選択肢2: 「いいえ（そのまま続ける）」
- **状態の変更**:
  - 「はい」を選択した場合、`app.js` の `totalSets` または `repsPerSet` の現在値を安全に変更し、UIを更新する。

## 3. 非機能要件

- **パフォーマンス**: 毎回の `tick` ではなく、セット完了時（`nextRepOrSet` 内の特定のタイミング）にのみ計算を行うため、タイマーの精度に影響を与えないこと。
- **ユーザビリティ**: 提案ダイアログは休憩時間（Phase.REST）中に表示されるため、ダイアログの操作を強制せず、休憩時間が終われば自動的に閉じる（または裏で消える）など、トレーニングの進行を阻害しない設計とする。今回は、モーダル表示中はタイマーを一時停止（Pause）する仕様がより安全かもしれないが、既存のポーズ機能との競合を避けるため、「休憩時間内に答える、答えなければそのまま」とする。
- **アクセシビリティ**: スクリーンリーダー対応（`aria-live` などを用いた音声通知、または `VoiceCoach` による読み上げ）。

## 4. 想定ユースケース

1. ユーザーが「3セット × 15回」のスクワットを開始する。
2. ユーザーは今日、仕事で疲れており、第1セットの途中で5秒間の一時停止を行い、さらにクイズにも連続で不正解する。
3. 第1セットが完了し、休憩時間（Phase.REST）に入る。
4. アプリが疲労を検知し、「コンディション・アシスト」モーダルを表示する。
   「今日は少しお疲れですか？ 無理せず、残りのセット数を2セットに変更しましょうか？」
5. ユーザーが「はい」を選択する。
6. 残りの目標が「2セット × 15回」に自動修正され、ユーザーは無理なくトレーニングを完了できる。

## 5. 入出力仕様

**入力（評価関数への引数）**:
- `sessionState`: 現在のセッション情報（`currentSet`, `totalSets`, `repsPerSet`）。
- `performanceData`:
  - `cumulativePauseDuration` (ms): これまでのポーズ合計時間。
  - `quizSessionCorrect` / `quizSessionTotal`: クイズの成績。
  - `ghostDiff`: ゴーストマネージャーから取得した遅延率。

**出力（戻り値）**:
- `shouldAssist` (boolean): アシストを提案するかどうか。
- `proposedAdjustment` (object): 提案する新しい設定内容（例: `{ newTotalSets: 2 }`）。

## 6. エッジケース

- すでに1セットしかない設定の場合（アシスト不要）。
- 提案ダイアログ表示中に休憩時間が終了し、次のセットのカウントダウンが始まってしまった場合（自動的に提案を取り下げるか、提案中はタイマーをポーズするか。今回は、**提案中はタイマーを自動的にポーズし、ユーザーの選択後に再開する**設計とする）。
- クイズ設定をOFF（出題なし）にしている場合（クイズの正答率はスコア計算から除外する）。

## 7. 失敗時の挙動

- コンディション計算に失敗した場合（エラー発生時）は、何も表示せず、通常のトレーニングフローを継続する（機能の非侵入性）。
