# Smart Mission Routine (ミッション最適化メニュー自動生成)

## 1. 目的

ユーザーがアプリを開いた際、「今日はデイリーミッションとビンゴをクリアするために、何セット・何回やればいいのか」をユーザー自身が考えて手動設定する認知負荷を減らすこと。
未達成のミッション・ビンゴから必要な「セット数」と「合計回数」を自動算出し、1クリックで設定入力欄に反映させる。

## 2. 機能要件

- **未達成分の抽出:**
  - `DailyMissionSystem.state.missions` の中から、未達成(`!completed`)の `total_reps` (合計回数) および `total_sets` (合計セット数) を抽出する。
  - `BingoManager.state.cells` の中から、未達成(`!completed`)の `total_reps` を抽出する。
- **必要数の算出:**
  - 必要な合計回数（Target Reps） = (Dailyの未達成 `total_reps` の残り) と (Bingoの未達成 `total_reps` の残り) のうち、**最大値**。
  - 必要な合計セット数（Target Sets） = (Dailyの未達成 `total_sets` の残り)。存在しなければデフォルト値（1セット等、もしくは回数から算出される最小セット数）。
- **最適メニューの計算ロジック:**
  - `Target Reps` と `Target Sets` を満たすように、`set-count` と `rep-count` (1セットあたりの回数) を決定する。
  - 基本戦略:
    - `Target Sets` が指定されている場合: セット数はそれに合わせる（最小1）。1セットあたりの回数は `Math.ceil(Target Reps / Target Sets)` とする。
    - `Target Sets` が指定されていない（0の）場合: デフォルトで3セットとし、1セットあたりの回数を `Math.ceil(Target Reps / 3)` とする。ただし、1セットあたりの回数が非常に少なくなる場合（例：合計5回）は、1セット5回等に調整する。
    - 算出された1セットの回数が大きすぎる場合（例えば50回を超えるなど）は、セット数を増やして1セットの回数を減らすよう補正する（最大 `rep-count` の上限などに配慮）。
  - いずれも必要な条件（合計回数、セット数）を**必ず満たす**最小の組み合わせとする。
  - もしミッションが全て達成済みの場合は、デフォルトの設定（3セット×10回など）を提案、もしくはトーストで「全て達成済みです」と通知し変更しない。
- **UI連携:**
  - UI上の「🎯 ミッション一括達成」ボタンをクリックした際に上記ロジックを実行する。
  - `app.js` の `setCountInput.value` および `repCountInput.value` に算出した値を代入し、関連イベント(`input`, `change`)を発火させてUIを更新する。
  - ユーザーへ「ミッションに最適化しました：Xセット × Y回」とToastで通知する。

## 3. 非機能要件

- 既存の `DailyMissionSystem` と `BingoManager` の設計（`state` プロパティでのアクセス）に直接依存し、追加のAPIやステート管理は導入しない。
- ローカルデータ（LocalStorage）に保存されている現在の進捗を正しく読み取り、計算に用いること。

## 4. 想定ユースケース

- **ケース1: 両方残っている場合**
  - Daily: 合計3セット必要、合計50回必要。
  - Bingo: 合計80回必要、現在20回完了（残り60回）。
  - 算出: 必要セット数=3、必要合計回数=Max(50, 60)=60。
  - 結果: 3セット × 20回 (= 合計60回) となる。
- **ケース2: Bingoの回数だけが残っている場合**
  - Daily: 全て完了。
  - Bingo: 合計100回必要、現在0回。
  - 算出: 必要セット数=指定なし(デフォルト3)、必要合計回数=100。
  - 結果: 3セット × 34回 (= 合計102回) または 4セット × 25回 (= 合計100回) など、常識的な範囲で分割される。
- **ケース3: 全て完了している場合**
  - 何も設定を変更せず、「全てのミッションを達成済みです！」とToastで通知する。

## 5. 入出力仕様

- **モジュール名:** `SmartMissionRoutine`
- **関数 `calculateOptimalRoutine(dailyState, bingoState)`**
  - 入力:
    - `dailyState`: `DailyMissionSystem.state` オブジェクト
    - `bingoState`: `BingoManager.state` オブジェクト
  - 出力: `{ sets: number, reps: number } | null`
    - (すべて完了済みの場合は `null` を返す)
- **関数 `apply()`**
  - 依存: `DailyMissionSystem.state`, `BingoManager.state`, DOM(`set-count`, `rep-count`)
  - 処理:
    1. `calculateOptimalRoutine()` を呼び出す。
    2. 結果が `null` なら「全て達成済みです」と通知。
    3. 結果があれば、DOMの値を更新し、イベントを発火させる。
    4. 「ミッションに最適化しました：[sets]セット × [reps]回」と通知する。

## 6. エラー・失敗時の挙動

- `DailyMissionSystem` や `BingoManager` の `state` が未初期化、または不正な形式の場合は安全に `null` を返し処理を中断（またはデフォルト設定を適用）する。
- 算出された回数やセット数がHTMLの `min` / `max` 属性（例：セット数1〜10、回数1〜50）を逸脱する場合は、上限/下限値でクリップ（制限）する。
