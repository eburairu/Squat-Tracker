# Spec-019: エンドレス・タワー (Endless Tower)

## 1. 概要
通常のワークアウトとは異なる、ゲーム性の高い「エンドレス・タワー」モードを追加する。
プレイヤーは体力が尽きるまで階層（フロア）を登り続け、最高到達階層（ハイスコア）を競う。

## 2. 目的
- **マンネリ解消**: 日々のルーチンワークとは異なる刺激を提供する。
- **限界挑戦**: 自分の限界に挑む楽しさを提供し、モチベーションを高める。
- **短時間プレイ**: 1階層ごとの区切りがあるため、少しの時間でも遊べる。

## 3. 基本ルール

### 3.1 進行形式
- **1階層 = 1セット**: 各階層は固定のスクワットセット（例: 10回）で構成される。
- **敵の強さ**: 階層が上がるごとに敵のHPが増加する。
  - `EnemyHP = BaseHP (100) + (Floor * 15)`
- **プレイヤー攻撃**: 通常通り、スクワット回数とステータス（攻撃力）でダメージを与える。

### 3.2 ダメージと回復
- **プレイヤーHP**: 初期値 `100`。上限 `100`。
- **被ダメージ**: 1セット終了時、**敵の残りHP割合** に応じてダメージを受ける。
  - `Damage = Floor( (EnemyCurrentHP / EnemyMaxHP) * 20 )`
  - 例: 敵HPを半分まで削った場合、10ダメージ受ける。倒しきれば（HP 0）、0ダメージ。
- **回復**: 階層クリア時、HPが `10` 回復する（上限 `100`）。
- **ゲームオーバー**: プレイヤーHPが `0` 以下になった時点で終了。

### 3.3 報酬
- 到達した階層を「ハイスコア」として記録する。

## 4. UI/UX仕様

### 4.1 エントリーポイント
- メイン画面（ダッシュボード）に「エンドレス・タワー」カードを追加。
  - 現在のハイスコアを表示。
  - 「挑戦する」ボタンで開始。

### 4.2 プレイ画面 (Tower Mode)
- **専用カード (`#tower-card`)**:
  - タワーモード開始時、通常の `#boss-card` を非表示にし、代わりに表示する。
  - **階層表示**: 「Floor {floor}」
  - **プレイヤーHP**: プログレスバーと数値で表示。
  - **敵情報**: タワー専用の敵（「タワーの番人」など）を表示。
  - **終了ボタン**: 「リタイア」ボタンで強制終了可能。

### 4.3 リザルト
- ゲームオーバー時、トーストまたはモーダルで結果を表示。
- ハイスコア更新時は演出を入れる。

## 5. データ構造

### 5.1 TowerState (メモリ内)
```javascript
{
  currentFloor: 1,
  playerHp: 100,
  maxPlayerHp: 100,
  enemy: {
    maxHp: 100,
    currentHp: 100
  },
  isActive: false
}
```

### 5.2 保存データ (localStorage)
- Key: `squat-tracker-tower-highscore`
- Value: `Number` (例: `15`)

## 6. 実装計画

### 6.1 TowerManager (js/modules/tower-manager.js)
- シングルトンモジュール。
- `init()`: イベントリスナー登録、ハイスコア読み込み。
- `startTower()`: モード開始、UI切り替え、初期化。
- `endTower()`: モード終了、UI復帰、スコア保存。
- `progressFloor()`: 次の階層へ。
- `calculateDamage()`: プレイヤー被ダメージ計算。

### 6.2 UI実装
- `index.html`:
  - `#tower-entry-card`: エントリー用カード。
  - `#tower-card`: プレイ用カード（初期非表示）。
- `styles.css`: タワー関連のスタイル。

### 6.3 app.js連携
- `finishWorkout` 内でタワーモード中か判定し、専用処理 (`TowerManager.onSetFinished`) を呼び出す。
- タワーモード中は通常の `BossBattle.damage` を呼ばないようにガードする（あるいは `TowerManager` がダメージ計算後に `BossBattle` の代わりに表示更新を行う）。
  - ※ `performAttack` 関数内で `TowerManager.isActive` をチェックし、分岐させるのが良い。

## 7. テスト計画
- **Unit Test**: `TowerManager` のロジック（ダメージ計算、HP管理）をテスト。
- **E2E Test**:
  - カードが表示されていること。
  - 「挑戦する」で専用UIに切り替わること。
  - スクワットで敵HPが減ること。
  - セット終了時にプレイヤーHPが変動すること。
  - 終了して通常画面に戻ること。
