# Spec-013: Squat Bingo (スクワット・ビンゴ)

## 1. 概要
ウィークリーチャレンジを、リスト形式から「3x3のビンゴ形式」に刷新する。
ユーザーはマスのミッションを達成してパネルを開け、縦・横・斜めのラインを揃えることで追加報酬を獲得する。
視覚的な達成感とゲーム性を高め、継続的なトレーニングを促進することを目的とする。

## 2. 目的
- **継続率向上:** 「あと1つでビンゴ」という動機づけにより、週後半の離脱を防ぐ。
- **ゲーム性強化:** 単調なタスク消化ではなく、戦略的にミッションを進める楽しさを提供する。
- **視覚的報酬:** 達成状況が一目でわかるUIにより、満足度を高める。

## 3. 仕様詳細

### 3.1 ビンゴカード構造
- **サイズ:** 3x3 (計9マス)
- **更新:** 毎週月曜日にリセットされ、新しいカードが生成される。
- **中央マス:** フリーマス（最初から達成済み）または簡単なミッションとする。今回は「簡単なミッション（ログイン1回など）」または「ラッキーマス（即時達成）」とするが、仕様簡略化のため**「通常のミッションマス」**として扱う。

### 3.2 データ構造 (LocalStorage: `squat-tracker-bingo`)
```json
{
  "weekId": "2024-W22", // 年-週番号
  "cells": [
    {
      "id": 0, // 0-8 のインデックス
      "type": "total_reps", // ミッションタイプ
      "target": 100, // 目標値
      "current": 50, // 現在値
      "completed": false, // 達成フラグ
      "claimed": false // (マス単体の報酬がある場合)
    },
    ...
  ],
  "lines": {
    "row1": false, "row2": false, "row3": false,
    "col1": false, "col2": false, "col3": false,
    "diag1": false, "diag2": false
  },
  "completed": false // 全達成コンプリートフラグ
}
```

### 3.3 ミッションタイプ
以下のタイプからランダムに配置される（重複可、ただしバランス調整推奨）。

| タイプ | ID | 説明 | 目標値例 |
| --- | --- | --- | --- |
| 総回数 | `total_reps` | 期間中のスクワット合計回数 | 100, 200, 300 |
| ログイン | `login_days` | アプリを開いた日数 | 3, 5 |
| ボス討伐 | `boss_kills` | 倒したボスの数 | 1, 3, 5 |
| クリティカル | `critical_hits` | クリティカルヒット発生回数 | 10, 30, 50 |
| プレイタイム | `total_seconds` | スクワット（フェーズ）総時間(秒) | 300, 600 |
| カロリー | `total_calories` | 消費カロリー (0.5kcal/rep換算) | 100, 200 |

### 3.4 報酬ロジック
- **マス達成報酬:** なし（ライン達成のための布石）。ただし、達成時に「スタンプ」が押される演出を入れる。
- **ライン達成報酬:** 縦・横・斜めのいずれかが揃うたびに報酬を獲得。
    - **報酬内容:** 武器（ランダム）、経験値、または称号キーワード。
    - **演出:** ラインが光り、Toastで報酬獲得を通知。
- **コンプリート報酬:** 8ライン全て（全マス）達成で、高レアリティ確定武器などの特別報酬。

### 3.5 画面UI (`index.html`)
- 既存の「ウィークリー」タブ内のリスト(`ul`)を削除し、`div.bingo-grid` に置き換える。
- 各セルは `div.bingo-cell`。
    - 未達成: ミッションアイコンと進捗バーを表示。
    - 達成済: 「済」スタンプやチェックマークをオーバーレイ表示。
- ライン達成時は、CSSアニメーションで該当ラインを強調表示する。

### 3.6 エッジケース処理
- **週の切り替わり:** アプリ起動時に現在時刻と `weekId` を比較し、異なればデータをリセットして新規生成する。
- **データ破損:** `localStorage` のデータが不正な場合は、新規生成して上書きする。
- **オフライン:** ローカルで完結するため問題なし。

### 3.7 既存機能との兼ね合い
- `WeeklyChallengeSystem` (`js/modules/weekly-challenge.js`) は廃止し、`BingoManager` (`js/modules/bingo-manager.js`) に置き換える。
- `DailyMissionSystem` はそのまま維持する。

## 4. テスト方針
- **Playwright E2Eテスト:**
    - ビンゴカードが正しく描画されるか。
    - スクワット完了時に進捗が反映されるか。
    - ライン達成時にToastが表示されるか。
    - 週が変わった時にリセットされるか（`page.clock.setSystemTime`等を使用）。
