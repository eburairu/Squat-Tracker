# general-task-playbook

## Description
非開発タスク（調査・ドキュメント作成・運用作業など）を抜け漏れなく進めるための計画・実行テンプレートを提供する。
Trigger examples: "調査タスク", "ドキュメント作成計画", "運用手順", "タスク整理", "task plan", "how to"

## 手順
1. **ゴール定義**: 何を達成したら完了とするかを明確にする。
   - 曖昧な場合はユーザーに確認する。
2. **前提整理**: 必要な情報、アクセス権限、参照ドキュメントを確認する。
3. **ステップ分解**: 作業を実行可能な粒度に分割する。
4. **検証方法**: 作業が正しく完了したことをどう確認するか決める。

## 出力フォーマット
```markdown
# タスク計画: <タスク名>

## 目的・ゴール
<達成すべき状態>

## 作業ステップ
1. [ ] <ステップ1>
2. [ ] <ステップ2>
   - 詳細: ...

## 検証・確認
- <確認方法>

## 実行ログ (随時更新)
- <実施したこと、結果、気づき>
```
