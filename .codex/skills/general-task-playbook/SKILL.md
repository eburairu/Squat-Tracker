# general-task-playbook

## 目的
汎用タスクを抜け漏れなく進め、結果を再現しやすくする。

## 手順
1. 目的と完了条件を1〜2文で要約する。
2. 影響範囲（ファイル/機能/ユーザー影響）を整理する。
3. 実行すべき作業を小さなステップに分解する。
4. 変更後の確認方法（テスト/動作確認）を定義する。
5. 実施結果と残課題をまとめる。

## 出力フォーマット
- 目的: <短い要約>
- 完了条件:
  - [ ] 条件1
  - [ ] 条件2
- 作業ステップ:
  1. ステップ1
  2. ステップ2
- 確認方法:
  - <テスト/チェック>
- 結果:
  - 実施内容の要約
  - 残課題（あれば）
