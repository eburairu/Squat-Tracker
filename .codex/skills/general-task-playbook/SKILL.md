# general-task-playbook

## Description
非開発タスク（調査・ドキュメント作成・運用作業など）を抜け漏れなく進めるための計画・実行テンプレートを提供する。
※機能実装や改修には `sdd-*` 系スキルを使用すること。
Trigger examples: "タスクの進め方教えて", "計画立てて", "調査手順", "運用手順書", "task playbook", "リサーチ計画", "ドキュメント作成計画", "汎用タスク"

## 手順
1. 目的と完了条件を明確にする。
   - ユーザーの意図が曖昧な場合は、作業前に質問してゴールを確定させる。
2. 影響範囲（ファイル/機能/ユーザー影響）を整理する。
3. 実行すべき作業を小さなステップに分解する。
4. 変更後の確認方法（テスト/動作確認）を定義する。
5. 実施結果と残課題をまとめる。

## 出力フォーマット
```markdown
# タスク計画

## 目的
<短い要約>

## 完了条件
- [ ] <条件1>
- [ ] <条件2>

## 作業ステップ
1. <ステップ1>
2. <ステップ2>

## 確認方法
- <テストコマンドや手順>

## 結果 (実行後記入)
- 実施内容: <要約>
- 残課題: <あれば記述>
```
