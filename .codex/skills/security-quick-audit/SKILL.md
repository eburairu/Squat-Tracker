# security-quick-audit

## 目的
変更差分に対する簡易セキュリティ監査を行う。

## 手順
1. 変更ファイルと影響範囲を列挙する。
2. 入力値の取り扱い（検証/サニタイズ）を確認する。
3. 認証/認可の境界に影響がないか確認する。
4. 依存関係の追加・変更があれば脆弱性リスクを確認する。
5. 機密情報（キー/トークン/PII）露出がないか確認する。

## 出力フォーマット
- 対象: 変更ファイル/機能の一覧
- 監査結果:
  - 入力検証: ✅/⚠️/❌ と短い理由
  - 認証/認可: ✅/⚠️/❌ と短い理由
  - 依存関係: ✅/⚠️/❌ と短い理由
  - 機密情報: ✅/⚠️/❌ と短い理由
- フォローアップ: 箇条書き（必要時）
