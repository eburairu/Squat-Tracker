# security-quick-audit

## Description
変更差分に対する簡易セキュリティ監査を行い、リスクを洗い出す。
Trigger examples: "脆弱性チェック", "セキュリティ診断", "security audit", "リスク評価", "脆弱性診断"

## 手順
1. 変更されたコードを取得する (`git diff` 等)。
2. 以下の重点項目を確認する:
   - **インジェクション**: SQL, Command, Code Injection の可能性。
   - **XSS**: ユーザー入力のエスケープ漏れ。
   - **認証/認可**: 権限チェックの不備、IDOR (Insecure Direct Object References)。
   - **機密情報**: APIキー、トークン、パスワードのハードコード。
   - **依存関係**: 新規追加されたパッケージの信頼性。

## 出力フォーマット
```markdown
# セキュリティ簡易監査

## 対象範囲
<確認したファイル/機能>

## 監査結果
- **インジェクション**: ✅/⚠️/❌ <コメント>
- **XSS**: ✅/⚠️/❌ <コメント>
- **認証/認可**: ✅/⚠️/❌ <コメント>
- **機密情報**: ✅/⚠️/❌ <コメント>
- **依存関係**: ✅/⚠️/❌ <コメント>

## 検出されたリスク
- **高**: <リスク内容>
- **中**: <リスク内容>
- **低**: <リスク内容>
```
