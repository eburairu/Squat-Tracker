# pr-review-checklist

## Description
Pull Request (または作成前のブランチ) のレビュー観点を標準化し、見落としを防ぐためのチェックリストを提供する。
Trigger examples: "レビューして", "PRチェック", "コードレビュー", "差分確認", "review PR", "check changes"

## 手順
1. 差分を取得する。
   - PR作成済みの場合: `git diff main...HEAD` (またはマージ先ブランチとの差分)
   - 作業中の場合: `git diff` (未ステージ) または `git diff --staged`
2. 以下の観点でコードを分析する:
   - **仕様整合性**: 要件を満たしているか、不要な変更がないか。
   - **安全性**: 入力検証、権限チェック、機密情報の漏洩がないか。
   - **堅牢性**: エラーハンドリング、境界値テスト、ログ出力。
   - **品質**: 可読性、命名規則、不要なコメント/デバッグコードの有無。
   - **テスト**: テストコードが含まれているか、カバレッジは十分か。

## 出力フォーマット
```markdown
# レビュー結果

## 概要
<全体の所感と推奨アクション（Approve / Request Changes / Comment）>

## チェック詳細
- **仕様整合性**: ✅/⚠️/❌ <コメント>
- **安全性**: ✅/⚠️/❌ <コメント>
- **堅牢性**: ✅/⚠️/❌ <コメント>
- **品質**: ✅/⚠️/❌ <コメント>
- **テスト**: ✅/⚠️/❌ <コメント>

## 改善提案 / 指摘事項
- [ファイル名:行番号] <指摘内容>
```
