# pr-review-checklist

## 目的
PRレビュー観点の標準化を行い、見落としを減らす。

## 手順
1. 変更されたファイル一覧を確認する。
2. 仕様・要件との整合性を確認する。
3. セキュリティ・権限・入力バリデーションの観点を確認する。
4. エラー処理・境界条件・ログ/監視の観点を確認する。
5. パフォーマンス・アクセシビリティ・UI/UXの影響を確認する（該当時）。
6. テスト/検証の有無と妥当性を確認する。

## 出力フォーマット
- 概要: 1〜3文でレビュー結果を要約
- チェック結果:
  - 仕様整合性: ✅/⚠️/❌ と短い理由
  - セキュリティ: ✅/⚠️/❌ と短い理由
  - エラー処理: ✅/⚠️/❌ と短い理由
  - パフォーマンス: ✅/⚠️/❌ と短い理由
  - UI/UX: ✅/⚠️/❌ と短い理由
  - テスト: ✅/⚠️/❌ と短い理由
- 追記/懸念: 箇条書き（必要時）
